## 3. 当前功能概述

PANDAyoloSight 是一个**纯批量图片分类工具**。用户选择一个包含图片的源文件夹和一个输出目录，系统自动对每张图片运行 YOLOv8 目标检测，按分类策略归入对应子目录。

**分类策略**：最高置信度主导——每张图片中置信度最高的物体决定该图片归属的类别。未检测到物体的图片归入"未分类"目录。

**核心交互流程**：

```
点击「批量分类」
    → 选择源文件夹 + 输出目录
    → 确认
    → 后台批量检测 + 分类（进度条实时反馈）
    → 右侧缩略图网格显示标注预览
    → 底部表格显示每张图的分类结果
    → 点击缩略图 → 弹窗大图查看（可缩放平移）
```

**输出规则**：
- 输出到文件夹的是**干净原图**（无边框无标签）
- 应用内缩略图和大图**叠加标注信息**（边界框 + 类别名）
- 目录结构按类别名自动创建子文件夹

## 5. 核心功能实现

### 5.1 模型管理 (ModelManager)

```java
// Criteria 构建 — DJL 模型动物园 + ONNX Runtime 引擎
Criteria.builder()
    .setTypes(Image.class, DetectedObjects.class)
    .optApplication(Application.CV.OBJECT_DETECTION)
    .optArtifactId("yolov8n")
    .optEngine("OnnxRuntime")
    .optOption("threshold", "0.10")   // ★ 关键: 键名是 "threshold" 不是 "optThreshold"
    .build();
```

- 首次加载从 DJL 仓库下载 yolov8n.onnx (~6MB) 到 `~/.djl.ai/cache/`
- 后续启动直接从本地缓存加载，无需网络
- `ZooModel` 线程安全全局共享；`Predictor` 每次新建并用 try-with-resources 关闭

### 5.2 推理管道 (YoloDetector)

```
BufferedImage (TYPE_3BYTE_BGR, 原始尺寸)
  → ImageFactory.getInstance().fromImage() → DJL Image
  → Predictor.predict() → DetectedObjects
  → 归一化坐标 [0,1] × 图片宽高 → 像素坐标
  → 按置信度降序排列 → List<DetectionResult>
```

YoloV8Translator 内部自动处理：letterbox resize 640×640、通道转换、归一化、NMS 后处理。

### 5.3 批量分类引擎 (BatchClassifier)

```java
public List<ClassificationResult> classifyFolder(
        File sourceDir, File outputDir, ProgressCallback onProgress)
```

**处理流程**：
```
遍历源文件夹中所有图片 (jpg/png/bmp/webp)
  │
  ├─ ImageUtils.loadImage(file) → BufferedImage
  ├─ YoloDetector.detect(img) → List<DetectionResult>
  │
  ├─ 分类决策: detections.get(0) (已按置信度降序)
  │     └─ primaryClass = best.className()
  │
  ├─ Files.copy(源, 输出目录/类别名/文件名)  ← 干净原图，不画框
  │
  └─ 返回 ClassificationResult (含完整检测数据，供缩略图渲染)
```

**边界情况**：
| 场景 | 处理 |
|------|------|
| 无检测结果 | 归入"未分类"目录 |
| 目标存在同名文件 | 自动追加 " (2)", " (3)" 后缀 |
| 处理异常 | 标记失败，继续下一张 |

### 5.4 分类结果 (ClassificationResult)

```java
public record ClassificationResult(
    String fileName,           // "photo001.jpg"
    String primaryClass,       // "person"
    float confidence,          // 0.936
    String sourcePath,         // 源文件路径
    String targetPath,         // 输出路径
    int totalDetections,       // 检测到几个物体
    boolean success,           // 是否成功
    List<DetectionResult> detections  // 完整检测数据（缩略图渲染用）
)
```

### 5.5 缩略图网格 (ThumbnailGridPanel)

- 使用 `FlowPane` 自动流式排列
- 每个缩略图 180×180px，在 `Canvas` 上渲染：原图缩放 + 边界框标注
- 缩略图标注数据来自 `ClassificationResult.detections`
- 颜色按 `className.hashCode() % 20` 从预定义色板分配
- 点击缩略图 → 弹出新 Stage，`ImageCanvasView` 全尺寸显示（支持缩放平移）

### 5.6 大图查看 (ImageCanvasView)

JavaFX Canvas 实现，核心能力：

**缩放**（鼠标滚轮，以光标为中心）：
```java
double scaleChange = zoomFactor / oldZoom;
panX = cursorX - scaleChange * (cursorX - panX);
panY = cursorY - scaleChange * (cursorY - panY);
```

**平移**（鼠标拖拽）：
```java
panX = panStartX + (mouseX - dragStartX);
panY = panStartY + (mouseY - dragStartY);
```

**缩放补偿渲染**（线条/字体始终清晰）：
```java
double invZoom = 1.0 / Math.max(zoom, 0.05);
double strokeW = 5.0 * invZoom;     // 基准 5px ÷ zoom
double fontSize = 20 * invZoom;      // 基准 20px ÷ zoom
```

### 5.7 批量结果面板 (BatchResultPanel)

- `TableView<ClassificationRow>`：文件名、分类、置信度、检测数、状态
- 置信度列颜色编码：绿 ≥80%、橙 50-80%、红 <50%
- "打开输出目录"按钮 → `Desktop.getDesktop().open(dir)`
- 进度条 + 分类统计 + 类别分布

### 5.8 线程模型

```
JavaFX Application Thread (UI)
  │
  └─ 用户点击「批量分类」
        │
        └─ new Thread(batchTask).start()  [后台线程]
              │
              ├─ call():
              │   ├─ modelManager.loadModel()
              │   ├─ classifier.classifyFolder()
              │   └─ updateProgress(cur, total)
              │
              ├─ progressProperty → Platform.runLater → sidebarProgress
              ├─ messageProperty  → Platform.runLater → batchResultPanel
              │
              ├─ setOnSucceeded()  [UI 线程]
              │   ├─ thumbnailGrid.populateFrom(results)
              │   ├─ batchResultPanel.setResults()
              │   └─ sidebarInfo.setText(stats)
              │
              └─ setOnFailed()  [UI 线程]
                    └─ Alert 弹窗
```

## 8. 部署架构

```
PANDAyoloSight.bat (桌面快捷方式)
    │
    ├─ JAVA_HOME → %USERPROFILE%\scoop\apps\openjdk17\current
    ├─ JavaFX → Maven 缓存中的 win JARs (module-path)
    ├─ ONNX Runtime → %USERPROFILE%\.yolosight\native\ (独立 DLL)
    ├─ YOLO 模型 → %USERPROFILE%\.djl.ai\cache\ (DJL 缓存)
    └─ App JAR → target\yolo-sight-1.0.0.jar (classpath)
```

启动脚本用 `--module-path` 加载 JavaFX、`-classpath` 加载应用 JAR，`-Donnxruntime.native.path` 指定独立 DLL。`%~dp0` 相对路径，无需配置环境变量。
