## 3. 核心功能实现

### 3.1 模型生命周期管理 (ModelManager)

**职责**：模型的下载、缓存、加载、销毁。

**类定义**：
```java
public class ModelManager implements AutoCloseable {
    private ZooModel<Image, DetectedObjects> model;
    private final Criteria<Image, DetectedObjects> criteria;
}
```

**模型选择策略**（Criteria 构建）：

```java
Criteria.builder()
    .setTypes(Image.class, DetectedObjects.class)      // 输入Image → 输出DetectedObjects
    .optApplication(Application.CV.OBJECT_DETECTION)   // 计算机视觉 · 目标检测
    .optArtifactId("yolov8n")                          // YOLOv8 nano 变体
    .optEngine("OnnxRuntime")                          // ONNX Runtime 后端
    .optOption("optThreshold", "0.5")                  // 默认置信度阈值
    .optOption("optNmsThreshold", "0.45")              // NMS IoU 阈值
    .build();
```

**加载流程**：

```
loadModel() 调用
    │
    ├─ model != null? ──→ 已加载，直接返回（幂等）
    │
    └─ model == null
         │
         ├─ DJL 检查 ~\.djl.ai\cache\ 本地缓存
         │    ├─ 命中 → 加载缓存模型（毫秒级）
         │    └─ 未命中 → HTTP 下载 yolov8n.onnx (~6MB)
         │         └─ 存入缓存 → 加载
         │
         └─ model = ZooModel 实例（线程安全，全局共享）
```

**Predictor 策略**：

```java
// 每次推理创建新 Predictor，用完即关
try (Predictor<Image, DetectedObjects> predictor = modelManager.newPredictor()) {
    detectedObjects = predictor.predict(djlImage);
}
```

原因：`Predictor` 不是线程安全的。`ZooModel` 是线程安全的，可以共享。

### 3.2 目标检测推理 (YoloDetector)

**职责**：将 Java 图片送入 YOLO 模型，输出结构化检测结果。

**核心方法签名**：
```java
public List<DetectionResult> detect(BufferedImage bufferedImage) throws Exception
```

**完整推理流水线**：

```
输入: BufferedImage (TYPE_3BYTE_BGR, 原始尺寸)
  │
  ├─ 步骤1: BufferedImage → DJL Image
  │   ImageFactory.getInstance().fromImage(bufferedImage)
  │   内部完成: BGR通道确认、内存布局对齐
  │
  ├─ 步骤2: DJL Image → Predictor → DetectedObjects
  │   try (Predictor predictor = modelManager.newPredictor()) {
  │       detectedObjects = predictor.predict(djlImage);
  │   }
  │
  │   YoloV8Translator (DJL自动选择) 内部处理:
  │   ┌─────────────────────────────────────────┐
  │   │ 预处理:                                  │
  │   │   ① letterbox resize → 640×640          │
  │   │   ② BGR → RGB 通道转换                  │
  │   │   ③ 像素值归一化 [0,255] → [0.0,1.0]     │
  │   │   ④ HWC → CHW 维度重排                  │
  │   │   ⑤ 添加 batch 维度 (1,3,640,640)       │
  │   │                                         │
  │   │ 推理:                                    │
  │   │   ONNX Runtime 前向传播                  │
  │   │   输出: (1, 84, 8400) 原始张量           │
  │   │                                         │
  │   │ 后处理:                                  │
  │   │   ① 解析 8400 个候选框                   │
  │   │   ② 每个框: [cx,cy,w,h, conf, 80类分数]  │
  │   │   ③ 置信度过滤 (< 阈值则丢弃)            │
  │   │   ④ NMS 去重 (IoU > 0.45 则抑制)        │
  │   │   ⑤ 输出 DetectedObjects                │
  │   └─────────────────────────────────────────┘
  │
  └─ 步骤3: DetectedObjects → List<DetectionResult>
       for (Classification item : detectedObjects.items()) {
           if (item instanceof DetectedObject obj) {
               BoundingBox bbox = obj.getBoundingBox();
               Rectangle rect = bbox.getBounds();

               // ★ 关键: 归一化坐标 → 像素坐标
               int x  = (int)(rect.getX()      * imgW);  // 0.5985 × 1280 = 766
               int y  = (int)(rect.getY()      * imgH);  // 0.3982 × 1706 = 679
               int w  = (int)(rect.getWidth()  * imgW);  // 0.4015 × 1280 = 513
               int h  = (int)(rect.getHeight() * imgH);  // 0.3916 × 1706 = 668

               results.add(new DetectionResult(
                   item.getClassName(),         // "bowl"
                   (float)item.getProbability(), // 0.936
                   new Rectangle(x, y, w, h),   // 像素坐标
                   -1                           // COCO classId
               ));
           }
       }
       按置信度降序排列

输出: List<DetectionResult> (5个物体)
```

**为什么坐标需要反归一化**：DJL 的 YoloV8Translator 输出的 BoundingBox 坐标是相对于 640×640 输入尺寸归一化到 [0, 1] 的值，需要乘以原始图片尺寸恢复为像素坐标。

### 3.3 图片显示与缩放平移 (ImageCanvasView)

**职责**：在 JavaFX Canvas 上显示图片、叠加检测边框、支持缩放和平移。

**核心数据结构**：
```java
public class ImageCanvasView {
    private StackPane view;         // 外层容器
    private Canvas canvas;          // 画布
    private GraphicsContext gc;     // 绘图上下文

    private Image originalImage;               // 原始图片
    private List<DetectionResult> detections;  // 检测结果
    private DetectionResult highlighted;       // 高亮的那个

    private double zoomFactor = 1.0;   // 缩放因子 (0.1x ~ 10.0x)
    private double panX = 0.0;         // X 平移偏移
    private double panY = 0.0;         // Y 平移偏移
}
```

**缩放实现（鼠标滚轮，以光标为中心）**：

```java
view.setOnScroll(e -> {
    double oldZoom = zoomFactor;                      // 旧缩放
    double delta = e.getDeltaY();
    zoomFactor = (delta > 0)
        ? Math.min(zoomFactor * 1.1, 10.0)            // 放大
        : Math.max(zoomFactor / 1.1, 0.1);            // 缩小

    double scaleChange = zoomFactor / oldZoom;        // 缩放比例差
    panX = e.getX() - scaleChange * (e.getX() - panX); // 保持光标处不动
    panY = e.getY() - scaleChange * (e.getY() - panY);

    redraw();
});
```

**平移实现（鼠标拖拽）**：
```java
view.setOnMousePressed(e -> {
    dragStartX = e.getX();  dragStartY = e.getY();
    dragPanX = panX;         dragPanY = panY;         // 记录拖拽起点
});
view.setOnMouseDragged(e -> {
    panX = dragPanX + (e.getX() - dragStartX);
    panY = dragPanY + (e.getY() - dragStartY);
    redraw();
});
```

**适应窗口实现**：
```java
public void fitToWindow() {
    double scaleX = viewW / imageW;              // 宽度缩放比
    double scaleY = viewH / imageH;              // 高度缩放比
    zoomFactor = Math.min(scaleX, scaleY) * 0.9; // 取较小者，留10%边距
    panX = 0; panY = 0;
}
```

**重绘流水线**：
```
redraw() 调用
  │
  ├─ gc.clearRect() + fillRect() → 填充深色背景(#151518)
  │
  ├─ originalImage != null?
  │   ├─ Yes:
  │   │   gc.save()
  │   │   gc.translate(panX, panY)       ← 平移
  │   │   gc.scale(zoomFactor, zoomFactor) ← 缩放
  │   │   gc.drawImage(originalImage, 0, 0) ← 绘制图片
  │   │   drawDetectionsJavaFX()            ← 绘制检测框
  │   │   drawHighlightedJavaFX()           ← 绘制高亮框
  │   │   gc.restore()
  │   └─ No: 绘制占位文字
```

**边界框缩放补偿**：
```java
double invZoom = 1.0 / Math.max(zoom, 0.05);
double strokeW = 2.5 * invZoom;    // 基准线宽 2.5px ÷ zoom
                                    // zoom=0.3 → 2.5/0.3=8.3px 图像坐标
                                    // 在屏幕上渲染 ≈ 2.5px
double fontSize = 14 * invZoom;    // 同理
```

保证缩放时线条和文字始终清晰可见。

### 3.4 检测结果可视化 (DrawingUtils + ImageCanvasView)

**颜色分配算法**：
```java
// 20 种预定义颜色，基于类别名的 hashCode 取模
private static final Color[] CLASS_COLORS = {
    "#FF3B30", "#34C759", "#007AFF", "#FF9500", "#AF52DE", ...
};

Color getClassColor(String className) {
    int hash = Math.abs(className.hashCode());
    return Color.web(CLASS_COLORS[hash % CLASS_COLORS.length]);
}
```
特点：同一类别在不同检测中颜色一致，视觉稳定。

**边界框绘制**：
```java
// 每个检测结果
gc.setStroke(color);                    // 类别对应颜色
gc.setLineWidth(strokeW);               // 缩放补偿后的线宽
gc.strokeRect(x, y, width, height);     // 像素坐标矩形

// 标签背景
gc.setFill(new Color(0, 0, 0, 0.7));    // 70% 不透明度黑色
gc.fillRoundRect(labelX, labelY, labelW, labelH, arc, arc);

// 标签文字
gc.setFill(Color.WHITE);
gc.fillText("bowl (93.6%)", labelX + pad, labelY + fontSize);
```

**高亮联动**：点击结果表格某行 → `imageCanvas.setHighlighted(det)` → 重绘时加粗 5px 亮色边框。

### 3.5 图片加载与转换 (ImageUtils)

**加载流程**：

public static BufferedImage loadImage(File file) throws IOException {
    BufferedImage original = ImageIO.read(file);   // AWT 标准读取
    if (original == null) throw IOException(...);
    return convertToBGR(original);                  // 转换为 DJL 兼容格式

g.drawImage(source, 0, 0, null);    // 像素逐个转换

### 3.6 结果保存

**合成流程**：
```java
// 创建新画布（RGB 格式，兼容 JPEG）
BufferedImage rendered = new BufferedImage(imgW, imgH, BufferedImage.TYPE_INT_RGB);
Graphics2D g2d = rendered.createGraphics();

// 先画原图
g2d.drawImage(currentImage, 0, 0, null);

// 再画检测框（使用无缩放版本 zoomFactor=1.0，即高分辨率渲染）
DrawingUtils.drawDetections(g2d, lastDetections);

g2d.dispose();
ImageIO.write(rendered, "png", outputFile);
```

保存的是**原图分辨率**的结果图，框线清晰不受显示缩放影响。

## 4. 线程模型

```
JavaFX Application Thread (UI)
  │
  ├─ 用户点击"检测"
  │     │
  │     └─ 创建 Task<List<DetectionResult>>
  │           │
  │           └─ 提交到后台线程池
  │                 │
  │                 ├─ call()  [后台线程]
  │                 │   ├─ ModelManager.loadModel()
  │                 │   ├─ YoloDetector.detect()
  │                 │   └─ return results
  │                 │
  │                 ├─ setOnSucceeded()  [UI 线程]
  │                 │   ├─ imageCanvas.setDetections()
  │                 │   ├─ resultTablePanel.setResults()
  │                 │   └─ statusLabel.setText()
  │                 │
  │                 └─ setOnFailed()  [UI 线程]
  │                       └─ Alert 弹窗显示错误
```

**线程安全原则**：

| 对象 | 线程安全 | 策略 |
|------|----------|------|
| `ZooModel` | ✅ 线程安全 | DJL 保证，全局共享一个实例 |
| `Predictor` | ❌ 不安全 | 每次推理 `new` + `try-with-resources` |
| `BufferedImage` | ✅ 只读 | 推理期间不修改 |
| JavaFX 控件 | ❌ 必须 UI 线程 | `Task.setOnSucceeded()` 自动切回 UI 线程 |

## 8. 关键算法摘要

| 算法 | 位置 | 说明 |
|------|------|------|
| 归一化坐标 → 像素坐标 | `YoloDetector.detect()` | `px = norm × imgSize` |
| NMS 去重 | DJL YoloV8Translator | IoU > 0.45 的重复框被抑制 |
| 颜色分配 | `getClassColor()` | `hashCode % 20` 循环 20 色 |
| 缩放补偿 | `ImageCanvasView.drawDetectionsJavaFX()` | `strokeW = 2.5 / zoom` |
| 光标中心缩放 | `view.setOnScroll()` | `panNew = cursor - scale×(cursor - panOld)` |
| 适应窗口 | `fitToWindow()` | `zoom = min(viewW/imgW, viewH/imgH) × 0.9` |
| 拖拽平移 | `view.setOnMouseDragged()` | `pan = panStart + (mouseNow - mouseStart)` |
