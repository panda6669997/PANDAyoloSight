## 2. 产品概述

### 2.1 产品定位

PANDAyoloSight 是一款基于 YOLOv8 深度学习模型的桌面端批量图片分类软件。用户选择一个图片文件夹，系统自动对所有图片运行目标检测，按分类策略将图片归入不同类别子目录，并提供带标注的缩略图预览和全尺寸查看。

### 2.2 目标用户

- 需要对大量图片按物体类别自动分类的用户
- 计算机视觉学习者、研究人员
- 摄影师、图库管理者

### 2.3 技术路线

```
Java 17 + JavaFX GUI
    → DJL (Deep Java Library) — Java 深度学习框架
    → ONNX Runtime — 跨平台推理引擎
    → YOLOv8n — 预训练目标检测模型（COCO 80类）
```

## 3. 功能需求

### 3.1 批量分类

| 编号 | 功能名称 | 功能描述 | 优先级 |
|------|----------|----------|--------|
| F-01 | 文件夹选择 | 用户通过侧边栏按钮触发，分别选择源图片文件夹和输出目标文件夹 | 高 |
| F-02 | 批量检测 | 系统遍历源文件夹中所有支持的图片格式（JPG/JPEG/PNG/BMP/WebP），逐张运行 YOLOv8 推理 | 高 |
| F-03 | 模型自动下载 | 首次运行时自动从 DJL 仓库下载 YOLOv8n（约 6MB）至本地缓存，后续无需网络 | 高 |
| F-04 | 异步处理 | 所有检测和分类操作在后台线程执行，UI 保持响应 | 高 |
| F-05 | 进度反馈 | 侧边栏进度条和底部进度标签实时显示处理进度（当前张数/总张数） | 高 |
| F-06 | 进度条保持 | 进度条始终可见，完成后停在 100%，下次分类自动归零 | 中 |

### 3.2 分类策略

| 编号 | 功能名称 | 功能描述 | 优先级 |
|------|----------|----------|--------|
| F-07 | 最高置信度主导 | 每张图片取置信度最高的物体作为该图片的唯一分类依据 | 高 |
| F-08 | 干净输出 | 输出到分类文件夹的图片为原始图片，不带边界框、标签或置信度信息 | 高 |
| F-09 | 自动建目录 | 按类别名自动创建子文件夹，同一类别的图片归入同一目录 | 高 |
| F-10 | 未分类处理 | 未检测到任何物体的图片归入"未分类"目录 | 高 |
| F-11 | 文件名冲突 | 目标目录存在同名文件时自动追加序号后缀（如 "photo (2).jpg"） | 中 |

## 4. 非功能需求

### 4.1 性能

| 编号 | 指标 | 要求 |
|------|------|------|
| NF-01 | 冷启动推理 | 首次（含模型下载）30 秒内完成 |
| NF-02 | 热启动推理 | 模型缓存后单张图片 500ms 内 |
| NF-03 | 批量处理 | 100 张图片 2 分钟内完成（取决于图片大小和 CPU） |
| NF-04 | GUI 响应性 | 操作期间界面不冻结，进度实时更新 |
| NF-05 | 模型大小 | YOLOv8n ≤ 10MB |
| NF-06 | 内存占用 | 运行时 ≤ 512MB |
| NF-07 | 启动时间 | 桌面快捷方式启动 ≤ 3 秒 |

### 4.2 兼容性

| 编号 | 指标 | 要求 |
|------|------|------|
| NF-08 | 操作系统 | Windows 10/11 64 位 |
| NF-09 | JDK 版本 | ≥ JDK 17 |
| NF-10 | 图片格式 | JPG、JPEG、PNG、BMP、WebP |
| NF-11 | 推理引擎 | ONNX Runtime 1.20.0，DLL 独立部署 |

### 4.3 可用性

| 编号 | 指标 | 要求 |
|------|------|------|
| NF-12 | 界面语言 | 全中文 |
| NF-13 | 操作步骤 | 点击按钮 → 选源文件夹 → 选输出目录 → 确认，四步完成 |
